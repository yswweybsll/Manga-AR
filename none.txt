# Cursor 是什么？

Cursor 是一款集成了 GPT、Claude、Gemini 等先进大语言模型能力的智能代码编辑器。它的界面和使用方式与 VS Code 非常接近，因此如果你已经熟悉 VS Code，上手 Cursor 基本没有门槛。

从界面布局可以看出，Cursor 保留了 VS Code 的核心体验，包括插件安装、Python 解释器配置、远程服务器连接、设置管理等功能。很多 VS Code 插件也可以在 Cursor 中继续使用。

注册完成后，你会拥有一个专属账号。不同账号和套餐对应的模型调用额度不同，免费额度用完后，可以根据需要选择官方订阅方案。

如果需要将界面语言切换为简体中文，可以点击顶部命令框，输入 `>language`，然后选择对应语言进行配置。

## 2.2 内置模型

Cursor 内置了多种大语言模型，包括 GPT、Claude、Gemini 以及部分推理模型。你可以在右上角的模型设置中选择合适的模型来辅助编程。

日常开发中，代码生成、代码修改、错误排查和项目理解是最常用的场景。不同模型在响应速度、代码能力和上下文理解方面各有特点，可以根据任务复杂度灵活选择。

## 2.3 常用快捷键

Cursor 最常用的快捷键主要有以下几个：

- `Tab`：接受 AI 自动补全
- `Ctrl + K`：编辑或生成代码
- `Ctrl + L`：对代码、文件或整个项目进行问答，也可以辅助修改代码
- `Ctrl + I`：基于对话对整个项目进行跨文件编辑

### Tab：自动补全

当 Cursor 预测到你接下来可能要编写的代码时，会在编辑器中给出灰色提示。确认无误后，按下 `Tab` 即可接受补全内容。

这个功能适合用于补全重复代码、函数调用、变量名、条件判断和常见逻辑结构，可以显著提升编码效率。

### Ctrl + K：生成或修改代码

`Ctrl + K` 主要有两类使用方式：

1. 从零开始生成代码
2. 修改已有代码

你也可以选中整段代码，让 Cursor 帮你优化结构、补充必要注释、重构逻辑或修复问题。

#### 从零开始编写代码

在空白区域按下 `Ctrl + K`，唤出编辑框后选择模型，并输入你的需求。Cursor 会根据描述生成代码。生成完成后，可以点击 `Accept` 接受修改，也可以点击 `Reject` 放弃修改。

#### 修改已有代码

选中已有代码后按下 `Ctrl + K`，输入修改要求，例如“优化这段代码”“修复这个报错”“改成 TypeScript 写法”等。Cursor 会基于选中的代码进行编辑，你可以逐行查看并决定是否接受修改。

### Ctrl + L：智能问答与代码分析

`Ctrl + L` 是 Cursor 中非常强大的功能，可以用于代码解释、问题排查、需求分析和项目级问答。

它支持针对以下范围进行提问：

- 当前选中的代码
- 当前文件
- 指定文件
- 整个项目

如果想针对某个文件提问，可以按下 `Ctrl + L` 打开右侧对话框，然后输入 `@`，选择 `Files`，再选择对应文件进行提问。

例如你可以这样提问：

- 这段代码的作用是什么？
- 这个函数有没有潜在问题？
- 请帮我优化当前文件的结构。
- 请根据这个组件生成对应的测试用例。

## 2.4 项目的全自动开发

`Ctrl + I` 适合处理跨文件、项目级别的开发任务。它可以通过对话方式帮助你创建文件、修改多个文件、删除无用代码，并完成较复杂的功能开发。

例如，你可以让 Cursor 创建一个简单的贪吃蛇游戏。它会根据需求生成相关文件和代码，完成后你可以点击 `Accept all` 一次性应用所有修改。

不过，对于复杂项目，建议不要一次性给出过大的需求。更推荐将任务拆分成多个小步骤，例如：

1. 先生成项目结构
2. 再实现核心功能
3. 然后补充样式和交互
4. 最后进行测试和优化

这样更容易控制质量，也便于及时发现和修正问题。

## 2.5 将外部文档作为知识库进行问答

Cursor 支持将外部文档添加为知识库，用于辅助代码理解和开发。比如，你可以把框架文档、接口文档、项目规范或第三方库文档加入 Cursor，让它在回答问题时参考这些资料。

添加文档后，使用方式与引用文件类似：按下 `Ctrl + L` 打开对话框，输入 `@`，选择 `Docs`，然后选择已经添加的文档进行提问。

这种方式非常适合以下场景：

- 阅读大型项目文档
- 理解第三方库 API
- 根据接口文档生成请求代码
- 根据团队规范优化代码风格

## 2.6 配置 Rules for AI

经常编写 Prompt 的同学应该知道，系统提示词可以帮助大模型更准确地理解自己的角色、任务边界和用户习惯。

在 Cursor 中，可以通过设置里的 `Rules for AI` 添加自定义规则，让 AI 在回答和修改代码时遵循你的偏好。

示例规则如下：

```markdown
# 角色定义
你是一名经验丰富的产品经理和工程师，负责帮助用户完成产品设计与开发任务。

# 目标
主动理解用户需求，提供清晰、可执行、易维护的解决方案，并帮助项目顺利推进。

## 第一步：理解项目基础
优先阅读项目文档和现有代码结构。必要时，补充或更新项目说明文档。

## 第二步：明确用户需求
充分理解用户目标，必要时主动补充关键细节，并优先选择简单、稳定、可维护的实现方案。

### 编写代码请求
分析需求，选择合适技术方案，编写结构清晰、易于维护的代码。

### 解决代码问题
先理解现有逻辑，再定位问题原因，并在必要时与用户沟通确认修改方案。

## 第三步：项目总结与反思
任务完成后，简要总结修改内容、影响范围和后续建议。
```

## 3. 合理使用 Cursor

Cursor 的 AI 能力非常强，但不同模型和套餐通常会有调用额度限制。为了更高效地使用额度，建议注意以下几点：

1. **明确描述需求**：提问越清晰，AI 生成结果越准确，反复修改的次数也会更少。
2. **拆分复杂任务**：将大需求拆成小步骤，可以降低出错概率。
3. **优先引用上下文**：使用 `@Files`、`@Docs` 等方式提供相关文件或文档，能提升回答质量。
4. **及时审查代码**：AI 生成的代码需要人工检查，尤其是涉及安全、性能和业务逻辑的部分。
5. **选择合适套餐**：如果免费额度无法满足需求，可以根据使用频率选择官方订阅方案。

总体来说，Cursor 更像是一位随时待命的编程助手。它可以帮助你更快理解项目、生成代码、排查问题和完成重构，但最终的技术判断和代码质量仍然需要开发者自己把控。
